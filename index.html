Overview

XRD Peak Fitting Applet is a browser-based tool for analyzing conventional thin-film 2θ–ω X-ray diffraction scans. It preserves the simple single-file workflow of the SUPERMAG applets: open the HTML file in a modern browser, load a diffraction file, inspect the full scan on a log-intensity plot, choose a local fitting window, fit one or more peaks together with a local background, review the fit, and export both the fit trace and a peak-summary table.

The app is intentionally narrow in scope. It is not a full phase-identification or refinement package. It is meant for the common thin-film use case where the user already knows the likely substrate and film, wants to isolate one local region of the scan, fit the peak shape cleanly, and extract quantities such as center position, FWHM, Bragg d-spacing, and approximate Scherrer size.

Quick start: Load an XRD file → choose optional substrate and film overlays → define the fit range on the main log-intensity plot → select number of peaks, model, and background type → click Fit Selected Range → inspect the fit review plot → adjust parameters and refit if needed → export the overlay CSV and summary CSV.
Workflow

The app is organized as a simple four-stage workflow. The emphasis is on manual control and local fitting rather than automated indexing or full-pattern processing.

1 Load Data

Load a diffraction file by drag-and-drop or by tapping the file picker. The app supports the same basic raw loader style as the existing SUPERMAG applets, including Rigaku and generic text-based exports. The parsed scan is then plotted as intensity versus 2θ.

The full scan is displayed on a logarithmic intensity axis by default. To keep the display usable, any intensity values below 1e-2, including zero or negative values, are floored to 1e-2 for plotting only. The raw data are preserved internally for the fit calculations.

Intended scan type: The app is designed for standard symmetric 2θ–ω scans. It does not currently branch into rocking curves, reciprocal-space maps, pole figures, or reflectivity analysis.
2 Reference Overlays and Fit-Range Selection

Two dropdowns are provided for visual guidance only: Substrate and Film. When selected, the app overlays expected peak-position markers from a small internal reference library containing common SUPERMAG materials.

These overlays are advisory only. They do not constrain the fit, do not perform automatic indexing, and do not propagate into the reported fit parameters. Their purpose is simply to help the user see what peaks are expected in the selected region.

The user then defines the local fitting window on the main plot using the fit-range controls. The selected range should contain the peak or overlapping peaks of interest together with enough adjacent baseline to support a stable local background fit.

Recommended practice: Choose a range that is wide enough to include the full peak tails and some background on each side, but not so wide that unrelated peaks or strong substrate features begin to dominate the local fit.
3 Fit Setup

For the selected local window, choose the number of peaks and the peak model. The app supports one, two, or three peaks in the local region. This is intended to cover common cases such as an isolated film peak, a film peak overlapped with a substrate peak, or a weak shoulder adjacent to a stronger reflection.

The available model choices are:

  • Gaussian — useful for simple symmetric peaks dominated by Gaussian-like broadening.
  • Lorentzian — useful when tails are sharper and longer than a Gaussian profile.
  • Pseudo-Voigt — the default and generally preferred choice for routine thin-film XRD peak fitting.

The background is fitted simultaneously with the peak model. The default background is constant, with linear available when a local slope is visibly needed.

Local fit model: I(2θ) = B(2θ) + Σ P_i(2θ) Constant background: B(2θ) = c Linear background: B(2θ) = m(2θ) + c

Initial guesses are generated automatically from the selected window and the chosen number of peaks. These guesses are intended to seed the least-squares fit, not replace it.

4 Fit Review and Manual Refinement

After clicking Fit Selected Range, the app opens a fit-review panel showing only the selected local range. The review plot overlays the raw data, the total fitted curve, the local background, and the individual peak components.

The user can then manually adjust the fit parameters and refit from those values. This is useful when the automatic initial guesses are close but not ideal, especially for overlapping peaks or asymmetric local intensity distributions.

Typical adjustable parameters include peak center, amplitude or height, width, and the pseudo-Voigt mixing parameter when that model is selected. Background parameters are also editable. The standard workflow is:

  • inspect the first-pass fit
  • adjust parameters if needed
  • refit from the edited values
  • accept the fit and move to results/export
Fit philosophy: The app is designed as an interactive local fitting tool. The sliders and numeric controls are there to improve the fit seed and guide the optimizer, not to replace the least-squares fit with a manual trace overlay.
Physics & Calculation Details
Peak Position and Bragg d-Spacing

For each fitted peak, the app reports the fitted center in 2θ and calculates the corresponding interplanar spacing d using Bragg's law with the standard crystallographic convention n = 1.

Bragg law: λ = 2 d sinθ with θ = (2θ) / 2

The reported d-spacing is therefore:

d = λ / (2 sinθ)

The wavelength is user-editable and defaults to the Cu Kα value used in the app settings. The app does not attempt to infer crystal structure or assign Miller indices automatically; that interpretation remains with the user.

Peak Width and Scherrer Size

For each fitted peak, the app reports the fitted FWHM and uses it to estimate a Scherrer size. The Scherrer calculation uses K = 0.9.

Scherrer equation: D = K λ / (β cosθ) with K = 0.9

Here β is the peak broadening in radians derived from the fitted FWHM. In the current app, the Scherrer result is reported as an approximate, uncorrected size.

Scherrer note: The reported Scherrer value is approximate and does not include instrumental-broadening correction. It should be interpreted as a quick comparative metric rather than a definitive crystallite size.
Supported Line Shapes

The pseudo-Voigt model combines Gaussian-like and Lorentzian-like broadening in a single practical fitting form. For most thin-film XRD peaks in routine laboratory data, this is the preferred default.

Gaussian and Lorentzian remain available for cases where a simpler fixed line shape is desired or where a direct comparison between models is useful.

Reference Overlay Library

The app includes a small built-in peak-position library for common substrates and films used in SUPERMAG workflows. Each entry contains only a material name, expected 2θ peak positions, and associated hkl labels.

This library is intentionally limited. It is not a full searchable diffraction database, and it does not perform phase scoring or automatic indexing. It is a plotting aid only.

Data Format Requirements

The app is intended for common text-based XRD exports and follows the same lightweight parsing philosophy as the existing SUPERMAG single-file tools. It is suitable for scan files in which the first numeric column is 2θ and the second numeric column is intensity, or for supported vendor-specific export formats already handled by the loader.

Supported input styleExamples
Rigaku.ras, .asc
Bruker text-like exports.dql, .raw when readable as text export
Generic text tables.csv, .txt, .dat, .xy with two numeric columns

The app expects the diffraction scan itself, not an already processed fit export. If multiple metadata blocks are present, non-numeric lines are ignored until the numeric data region is reached.

Troubleshooting
  • The log plot looks unusable because the baseline is too low: The app floors displayed intensity values to 1e-2. If the view still looks odd, reset the zoom and verify the input file has sensible intensity values.
  • The wrong peak is being fitted: Recheck the selected fit range. The app fits only the local window you define, so a misplaced range is the most common cause of an unexpected result.
  • The fit is unstable for overlapping peaks: Increase the fit window slightly, try pseudo-Voigt, and use the fit-review controls to move the initial centers closer to the visible maxima before refitting.
  • The background looks overfit: Switch from linear to constant background if the local baseline is nearly flat. The default background model is constant for this reason.
  • The fit-review graph does not appear immediately: Click Fit Selected Range again after confirming the range is valid. If the issue persists, resize the browser or reload the applet and retry. This behavior is usually related to canvas rendering rather than the fit itself.
  • The Scherrer value looks too small or too large: Remember that the current result is uncorrected for instrumental broadening. Use it as a comparative indicator, not as a final microstructural value.
  • The file fails to parse: Check that the file is a text-based export and that the scan data appear as numerical 2θ/intensity columns.
